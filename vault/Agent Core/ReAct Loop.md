# ReAct Loop

The ReAct (Reason + Action) loop is the central execution pipeline of OpenManus, implemented in `ReActAgent` (defined in `app/agent/react.py`) which inherits from `BaseAgent` (in `app/agent/base.py`). This architecture maps to the classical ReAct paradigm where the model alternates between reasoning about the task and executing actions.

## Core Mechanisms
1. **Lifecycle Event Tracing**: The loop emits explicit lifecycle events at each transition, allowing the frontend client to display real-time progress:
   - `agent:lifecycle:step:reason` — The text thought block generated by the LLM.
   - `agent:lifecycle:step:act` — Dispatched before tool calls are executed.
   - `agent:lifecycle:step:observe` — Dispatched with the result/observation of the tool call.
   - `agent:lifecycle:phase` — Real-time phase tracking (`AgentPhase`).
2. **State Management**: Uses the Pydantic-based `BaseAgent` state machine to manage transitions between `IDLE`, `RUNNING`, `ERROR`, and `FINISHED` via the `state_context` manager.
3. **Loop Control**: The main execution loop is controlled by the `run` method in `BaseAgent` which iterates up to `max_steps` or until the agent state enters `FINISHED`.
4. **Stuck Detection**: Implements a double-signal algorithm in `is_stuck()` to detect infinite loops:
   - *Semantic tool-loop*: Checks if the exact same tool and arguments are executed three or more times in the last 12 messages.
   - *Content-hash*: Normalizes and hashes textual output to detect exact near-duplicates. When a loop is detected, it injects a strategy-change prompt.
